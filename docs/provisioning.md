# Provisioning Guide

This guide explains how ClawHost provisions OpenClaw instances and how to operate provisioning safely.

## Provisioning Flow

1. Validate request and cloud provider config.
2. Create server with selected instance type and region.
3. Inject cloud-init generated by ClawHost.
4. Install and configure OpenClaw automatically.
5. Register instance in ClawHost and expose status/metrics.

## Minimal Provisioning Inputs

- Provider (for example: Hetzner)
- Server type (for example: `cx11`)
- Location (for example: `nbg1`)
- OpenClaw LLM config
- Labels/tags for ownership and environment

After creating an instance, verify:

- Instance status endpoint returns `running`
- Health endpoint reports healthy checks
- Metrics endpoint returns CPU/memory values

## Operational Best Practices

- Use labels for environment (`dev`, `staging`, `prod`).
- Prefer smaller types for tests, larger for production.
- Keep provisioning idempotent and retry-safe.
- Track provisioning jobs and audit logs.
- Set timeouts for each provisioning stage.
